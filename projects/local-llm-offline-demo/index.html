Local LLMs · video demo

local-llm-offline-demo

A capable LLM, running fully offline on a Mac Studio M3 Ultra — no cloud, no API keys, no network. Recorded for WOŚP as a live answer to the question “what can you actually do without the internet?”

Recorded live · airplane-mode demo · YouTube mirror — watch on YouTube ↗

What you’re looking at

Why the demo exists, and what is — and isn’t — running locally.

01

The premise

Most “AI” people see day to day is a thin client over a remote API. Pull the plug and it’s gone. This demo flips that: Wi-Fi off, ethernet off, model still answers.

02

The hardware

Mac Studio M3 Ultra with 256 GB unified memory — RAM and VRAM are the same pool, so a quantized 70B+ dense (or 100B+ MoE) model fits comfortably and stays interactive.

03

The stack

A local inference runtime (llama.cpp / MLX class) loading an open-weights model with a quantized format. No bespoke training — the point is that off-the-shelf weights are already enough to be useful offline.

04

The point

A frame for the “local vs. cloud” conversation: privacy, latency, cost, sovereignty, and what the gap to frontier models actually feels like for everyday tasks.

Where this fits

Part of an ongoing thread on local-first LLM workflows on this machine.

The longer arc is a local-agent-stack — eval harness, tool-use scaffolding, and Claude as orchestrator on top of local open-weights models for the steps that don’t need a frontier model. This demo is the cold-open: proof that the substrate works, before any of the orchestration goes on top.
