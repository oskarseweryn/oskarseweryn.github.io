How it works, end-to-end

01

Environment

Custom Snake env in ~150 lines of Python. 12×12 grid, three relative actions (straight / turn right / turn left). Reward is +1 per food, −1 on death, with a small per-step penalty to discourage looping.

02

State features

11 hand-crafted binary features: three immediate-collision flags (front / right / left), a one-hot of current heading, and four food-relative direction flags. Same code path runs in Python during training and JS at inference time — identical numbers, by construction.

03

Algorithm

Vanilla DQN with a target network, replay buffer (100k transitions), Adam, Huber loss, ε-decay from 1.0 to 0.02 over 60k steps. ~10k gradient steps across 4k episodes is enough for the policy to ramp from random to consistent food-chasing.

04

Export & deploy

Trained on M3 Ultra in a few minutes. Final weights are dumped as a single JSON file (architecture + params), no ONNX runtime required. The browser does its own forward pass in plain JavaScript — this whole page is static, hosted on GitHub Pages.

Training code, env, and exported weights: github.com/oskarseweryn/oskarseweryn.github.io/projects/snake-rl ↗
